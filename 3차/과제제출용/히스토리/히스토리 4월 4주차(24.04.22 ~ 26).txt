240423 회의 
1. 성남시 데이터 공모전 참여유무
2. 크롤링 데이터 전처리 회의

-------------------------------------
- 성남시 데이터도 하다가 취업하는 사람이 빠지는걸로
------------------------------------
[분석 방향]

종속변수: 회귀모델과 분류모델을 1개씩 구성
- 회귀: 몇 명이나 다칠지 (종속변수)
- 분류: 부상의 정도 (종속 변수)
- 작업자 수(범위로 나와있음) 포함
- 지수말고 퍼센트로 1차 진행
(지수 사용시 관련 논문 찾기가 어려움)
- 부상자 + 사망자=> 사상자로 합치기

[추후 방향]
- 날씨: 맑음인데 90%습도인 경우도 있어서 기상청 데이터 사용 고려
- 크롤링 완료
- 수치나 범주형이 아닌 어떻게 가공할 것인지 텍스트 처리 해야함.

[컬럼 삭제]
- 구체적 사고원인 : 삭제 -> 사고원인의 구체적 사유이다보니 사고원인 컬럼만 사용해도 된다고 판단
- 사고원인 : 회귀분석이기에 이미 벌어진 원인과 경위는 삭제
- 사고경위 삭제
- 장소 못씀. 사고 원인과 마찬가지
- 날씨는 맑음인데 90%습도인 경우도 있어서 기상청 데이터 사용 고려
- 피해금액 삭제: 인명 사상사고 시 피해금액이 없는 걍우가 많음
- 재발방지대책
- 시군구
- 낙찰률
- 해체계획서
- 사고조사방법
- 위원회조사
- 위원회구성
- 향후조치계획

[컬럼 사용]
- 인적방호
- 공종
- 시설물종류는 사용
- 사고객체 => 대분류로 사용
- 작업프로세스 : 10개 이하는 기타로 묶음
- 장소 : 대분류만 사용
- 사고 부위: 대분류만 사용
- 사고객체 : 대분류로 그륩화하기
- 피해내용: 필요한데 텍스트라 전처리 방법 구상 후 진행
- 사고 신고사유: 항목을 라벨링
- 공사종류: 토목도 있어서 사용(대중소)
- 공사비: 데이터 2종류로 되어있어서 분리필요
- 공사기간: 공사비와 마찬가지로
- 공정률
- 설계안전성
- 안전관리 계획

+ 평일, 주말, 공휴일, 분기, 계절

- 마이크로 데이터는 정제되어있음
